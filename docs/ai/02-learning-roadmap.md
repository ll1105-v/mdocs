# 新手学习路线图

如果你刚入门，建议按“概念 -> 调用 -> 场景 -> 项目”的顺序学习。

## 第一阶段：建立基本认知

先弄明白下面几个词分别是什么意思：

- LLM
- Prompt
- Token
- Context Window
- Embedding
- RAG
- Agent

这一阶段的目标不是背定义，而是能大致说出它们在产品里分别承担什么作用。

## 第二阶段：学会调用模型

你至少要完成一次最简单的 API 调用，知道下面这些事情：

- 怎么传入用户输入
- 怎么设置系统提示词
- 怎么读取模型返回结果
- 怎么处理超时、限流和失败重试

这一步会帮你从“会聊天”转到“会开发”。

## 第三阶段：做一个单点功能

建议从下面这类项目里选一个：

- 文章总结器
- PDF 问答助手
- 提示词优化器
- 会议纪要整理工具
- 个人知识库问答页

目标是做一个单功能、能演示、能上线的最小产品。

## 第四阶段：补足关键能力

当你完成第一个项目后，再集中补下面几块：

- Prompt 设计
- 向量检索与 RAG
- 工具调用
- 输出结构化
- AI 应用评估

## 第五阶段：面向真实使用优化

这一步才开始考虑产品化问题：

- 成本控制
- 响应速度
- 结果稳定性
- 权限与数据安全
- 用户反馈闭环

## 一条适合个人开发者的路线

1. 先用现成 API 做出可运行 demo
2. 再加上文档检索或个人知识库
3. 再根据场景决定是否需要 Agent
4. 最后再考虑长期维护和部署

## 不要一开始就陷进去的坑

- 只看模型榜单，不做项目
- 一上来就想训练自己的模型
- 过早追求复杂架构
- 没有评估标准，只靠“感觉还行”

入门最有效的方法，永远是边学边做。
