# 常见技术栈与工具选择

AI 应用开发并没有唯一标准栈，关键是选一套自己能快速做出成果的组合。

## 一个适合初学者的基础栈

- 前端：HTML / Vue / React，任选熟悉的
- 后端：Node.js、Python 都可以
- 模型接口：第三方大模型 API
- 数据存储：SQLite、PostgreSQL、Supabase 等
- 部署：Vercel、Netlify、云服务器都可以

## 为什么不要一开始就堆很多框架

初学阶段最重要的是：

- 跑通调用链路
- 看见真实输入输出
- 观察用户体验问题

如果一开始就把注意力花在复杂框架、抽象层和编排系统上，反而容易失去对核心问题的把握。

## 常见工具类型

### 模型接入层

负责和模型 API 通信，处理消息、参数和返回结果。

### 向量检索层

当你做文档问答、知识库或 RAG 时，会需要分块、向量化、召回和重排。

### 工作流编排层

当你的应用开始涉及多步骤处理、多工具调用时，编排层会变得更重要。

### 观测与评估层

用来记录提示词版本、模型输出、错误率、用户反馈和成本数据。

## 给个人项目的建议

如果你现在只是想做一个简易个人网站，并逐步加入 AI 内容或 AI 功能，可以按这个顺序扩展：

1. 先用 Docsify 放文档和学习笔记
2. 再加上一个简单的 demo 页面
3. 再接入模型 API 做单点功能
4. 最后再考虑检索、账号系统和复杂流程

## 选型标准

工具不是越新越好，优先看这三点：

- 你是否能快速上手
- 文档是否清晰
- 是否方便后续维护

先做成，再做强，通常是更稳的路线。
