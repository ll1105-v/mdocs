# AI名词解释

这篇文档用来解释 AI 应用开发里最常见的一批名词。目标不是追求学术定义，而是帮助你从工程角度理解：这个词是什么、解决什么问题、在系统里通常放在哪一层。

## 1. AI 名词概览

AI 应用开发中最常见的核心名词有：

- LLM
- Token
- Context Window
- Prompt
- Embedding
- RAG
- Function Calling
- Tool Calling
- Agent
- Workflow
- MCP
- Skill

如果你刚开始学，建议先记住一条主线：

- `LLM` 负责理解和生成
- `Prompt` 负责告诉模型要做什么
- `Tool / Function Calling` 负责让模型调用外部能力
- `RAG` 负责给模型补充外部知识
- `Workflow` 负责把多个步骤串起来
- `Agent` 负责在更复杂任务里做决策和执行

## 2. 基础概念

### 2.1 AI

AI 是 Artificial Intelligence，也就是人工智能。

在今天的大多数应用开发场景里，大家说的 AI 往往不是泛指所有人工智能技术，而是更具体地指：

- 大语言模型能力
- 文本、图片、语音等生成能力
- 基于模型的问答、总结、提取、分类、推理能力

所以在工程语境里，“做 AI 应用”通常不是自己训练模型，而是把现成模型能力接进业务系统。

### 2.2 LLM

LLM 是 Large Language Model，也就是大语言模型。

你可以把它理解成一个非常强的“文本理解与生成引擎”。它擅长：

- 问答
- 总结
- 改写
- 分类
- 信息提取
- 代码生成
- 多轮对话

但它本身也有边界：

- 不天然知道你的私有数据
- 不保证每次都完全正确
- 不能直接访问数据库、接口、文件，除非你给它工具

工程上，LLM 通常是 AI 应用的核心能力提供者。

### 2.3 Token

Token 可以理解为模型处理文本时使用的“最小计量单位”。

它不完全等于“一个字”或“一个单词”，而是模型内部切分文本后的单位。你在调用模型时，很多能力都和 Token 有关：

- 输入长度限制
- 输出长度限制
- 调用成本
- 响应速度

工程上你要关心 Token，因为：

- Prompt 太长会增加成本
- 上下文太多会影响响应速度
- 检索结果塞太多会挤占有效上下文

### 2.4 Context Window

Context Window 是上下文窗口，指模型一次最多能处理多少上下文内容。

这部分上下文通常包括：

- 系统提示词
- 用户输入
- 历史对话
- 检索出来的知识片段
- 工具调用结果

如果总内容超过窗口限制，就会出现内容被截断、历史丢失、回答质量下降等问题。

所以从工程角度看，上下文窗口是 Prompt 设计、RAG 设计、聊天历史管理的硬约束。

### 2.5 Prompt

Prompt 就是你发给模型的提示词或指令。

它不只是“问一句话”，而是你对模型任务的完整描述，通常会包括：

- 角色设定
- 任务目标
- 规则约束
- 输出格式
- 示例
- 上下文数据

一个工程化 Prompt 往往不是随手写的，而是需要版本管理、反复调试、持续迭代。

### 2.6 Embedding

Embedding 是把文本转换成向量表示的过程或结果。

向量的作用不是给人读，而是让系统能计算“语义上像不像”。因此 Embedding 常用于：

- 语义检索
- 相似内容召回
- 知识库搜索
- 去重与聚类

在 RAG 里，Embedding 不是负责“回答问题”的模型，而是负责“找到相关资料”的模型。

## 3. 应用开发常用概念

### 3.1 RAG

RAG 是 Retrieval-Augmented Generation，通常翻译为“检索增强生成”。

它解决的问题是：模型本身不知道你的私有知识，或者知道得不准，怎么办。

RAG 的基本流程通常是：

1. 准备文档
2. 对文档切分
3. 生成 Embedding
4. 把向量存入向量库
5. 用户提问时先检索相关片段
6. 把片段和问题一起交给模型生成答案

它的核心思想不是“训练模型记住你的知识”，而是“回答前先查资料，再结合资料回答”。

适合用 RAG 的场景：

- 企业知识库问答
- 产品文档助手
- 内部制度问答
- FAQ 问答

### 3.2 Function Calling

Function Calling 指模型按约定格式输出“我要调用哪个函数、传什么参数”。

注意，这里的函数通常不是模型自己真的执行，而是：

1. 模型判断需要某个工具
2. 模型返回函数名和参数
3. 你的后端代码执行这个函数
4. 把执行结果再喂给模型
5. 模型组织最终回复

它的价值在于让模型输出更稳定的结构化调用意图，而不是让模型自己编造接口调用逻辑。

常见场景：

- 查天气
- 查数据库
- 调内部接口
- 创建工单
- 发送消息

### 3.3 Tool Calling

Tool Calling 和 Function Calling 在很多场景里可以近似理解。

更宽一点地说：

- Function Calling 更强调“按函数签名调用”
- Tool Calling 更强调“调用外部工具能力”

在工程实践里，工具可能是：

- 一个本地函数
- 一个内部 HTTP 接口
- 一个数据库查询能力
- 一个搜索能力
- 一个文件读写能力

所以你可以把 Function Calling 看成 Tool Calling 的一种常见实现方式。

### 3.4 Agent

Agent 一般翻译为智能体。

它不是单纯“一次模型调用”，而是一套更完整的执行机制，通常具备这些特征：

- 能理解目标
- 能拆分任务
- 能决定下一步做什么
- 能调用工具
- 能根据结果继续迭代
- 能在多步后产出最终结果

如果说普通调用更像“你问一句，我答一句”，那 Agent 更像“接到任务后，自己规划并逐步完成”。

但要注意，很多场景并不需要重型 Agent。能用固定流程解决的问题，通常先用 Workflow 更稳。

### 3.5 Workflow

Workflow 就是工作流，也就是把多个步骤按规则编排起来。

例如一个典型流程可以是：

1. 接收用户问题
2. 判断问题类型
3. 检索知识库
4. 调用模型生成答案
5. 做格式校验
6. 返回结果

Workflow 的特点是步骤相对确定、可控、可测试。

所以很多 AI 应用早期并不是先做 Agent，而是先做 Workflow。因为它更稳定、更容易排错，也更适合生产环境。

### 3.6 MCP

MCP 是 Model Context Protocol。

你可以把它理解成一种协议标准，用来把“模型”与“外部工具或外部上下文”连接起来。

它要解决的问题是：不同工具、不同数据源、不同宿主环境，怎样用更统一的方式给模型提供能力。

从工程角度看，MCP 更像基础设施层的协议，而不是一个具体业务功能。它常用于：

- 给模型提供文件访问能力
- 给模型提供数据库访问能力
- 给模型提供浏览器、命令行、知识库等工具能力

### 3.7 Skill

Skill 可以理解为一组可复用的能力封装。

它通常不只是一个 Prompt，而是一套更完整的组合，例如：

- 一段任务说明
- 一套操作规范
- 一组工具使用方式
- 某个领域的最佳实践

在 Agent 或 AI 编码助手场景里，Skill 往往用来复用特定任务的处理方法，比如：

- 代码评审
- 文档整理
- 特定框架开发
- 特定平台部署

## 4. 容易混淆的区别

### 4.1 Prompt 和 RAG 的区别

两者都在影响模型回答，但职责不同：

- Prompt 解决的是“怎么提要求”
- RAG 解决的是“怎么补知识”

Prompt 像写任务说明书，RAG 像先给模型找参考资料。

### 4.2 Embedding 和 LLM 的区别

两者都属于模型能力，但用途不同：

- LLM 负责理解、生成、推理、回答
- Embedding 负责把内容转成向量，方便检索和相似度计算

简单说，Embedding 更常用于“找”，LLM 更常用于“答”。

### 4.3 Function Calling 和 Agent 的区别

- Function Calling 是一次工具调用机制
- Agent 是围绕目标持续决策和执行的系统

也就是说，Agent 经常会用到 Function Calling，但 Function Calling 本身不等于 Agent。

### 4.4 Workflow 和 Agent 的区别

- Workflow 是预先设计好的流程
- Agent 是根据目标动态决定下一步

Workflow 更稳定、更可控；Agent 更灵活，但复杂度和不确定性也更高。

### 4.5 Tool 和 MCP 的区别

- Tool 是具体能力，比如搜索、读文件、查数据库
- MCP 是让这些能力能被模型更统一接入的一种协议方式

前者偏“能力本身”，后者偏“接入标准”。

## 5. 面向开发者的理解方式

如果你是后端开发者，可以先用下面这套方式记忆：

- `LLM`：核心推理与生成引擎
- `Prompt`：给模型的任务说明书
- `Token`：调用长度和成本单位
- `Context Window`：一次请求能装下多少上下文
- `Embedding`：用于语义检索的向量表示
- `RAG`：先检索资料，再让模型回答
- `Function Calling / Tool Calling`：让模型调用外部能力
- `Workflow`：把多步任务编排起来
- `Agent`：能自主做多步决策的执行体
- `MCP`：连接模型与外部能力的协议
- `Skill`：可复用的任务能力封装

## 6. 学习建议

如果你刚入门，不要试图一次把所有名词都学透。建议按下面顺序理解：

1. 先理解 `LLM`、`Prompt`、`Token`
2. 再理解 `Context Window`、`Embedding`
3. 然后学习 `RAG`
4. 再学习 `Function Calling` 和 `Workflow`
5. 最后理解 `Agent`、`MCP`、`Skill`

原因很简单：

- 前面的概念是基础
- 中间的是 AI 应用开发核心能力
- 后面的更偏系统化、工程化、平台化能力
